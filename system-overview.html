1. システム概要

本システムは、個人向け家計簿サービス MoneyForward ME から入出金明細・口座残高データを 自動スクレイピングし、CSV ファイルの出力および Notion データベースへの書き込みを ノーコードで実現する自動化パイプラインです。

Notion カスタムエージェント（Worker）から自然言語で取得指示を出すと、GitHub Actions 上でヘッドレスブラウザが起動し、 30〜90 秒以内にデータ収集・同期を完了します。2 段階認証（OTP）の自動突破にも対応しており、 人手を介さない完全自動運用が可能です。

Python 3.11 Playwright Notion API Notion Worker GitHub Actions Google Apps Script TypeScript

2. 主要機能

🤖 自動スクレイピング

Playwright でブラウザを制御し、MoneyForward ME にログインして入出金明細・口座残高を自動取得します。

🔐 OTP 自動突破

Google Apps Script が Gmail からワンタイムパスワードを自動読み取り、2 段階認証を無人で突破します。

💾 セッション永続化

GitHub Actions キャッシュにセッションを保存し、毎回のログインを省略。処理時間を大幅に短縮します。

📅 柔軟な期間指定

今月・先月・今週・先週・指定月・任意期間の 6 モードに対応。Notion エージェントから自然言語で指定可能。

🗄️ Notion Upsert

取引日・金額・口座名をキーに重複チェックを行い、既存レコードは更新、新規は追加する Upsert 処理を実装。

📊 リアルタイム進捗

スクレイピング中の進捗を Notion ジョブ履歴ページへ随時反映。処理状況をリアルタイムで確認できます。

5. コンポーネント詳細

5-1. Notion Worker（worker/src/index.ts）

Notion のカスタム AI エージェント機能である Notion Worker として動作する TypeScript モジュールです。

エージェントから mf-trigger-scraping ツールが呼び出されると、以下の 3 ステップを実行します。

  1. Notion API でジョブ実行履歴 DB にページを作成（状態: 実行中）
  2. GitHub API の repository_dispatch でスクレイピングワークフローを起動
  3. ジョブページ ID と「started」ステータスをエージェントへ返答
取得期間モード: 6 種類 入力バリデーション付き TypeScript / @notionhq/workers

5-2. MoneyForward スクレイパー（scraper/mf_scraper.py）

Playwright を使ってヘッドレス Chromium ブラウザを操作し、MoneyForward ME からデータを取得するコアモジュールです。

セッション管理

  • 初回ログイン後に session.json へクッキーを保存
  • 次回以降はセッションを再利用し、ログインをスキップ
  • セッション切れを自動検知して再ログイン

2段階認証（OTP）

  • GAS WebApp に 3 秒間隔で最大 10 回ポーリング
  • 6 桁コード取得後に自動入力・送信
  • GAS 未設定時は手動入力にフォールバック

5-3. Notion ライター（scraper/notion_writer.py）

取得したトランザクションデータを Notion 取引履歴 DB へ書き込むモジュールです。

Upsert ロジック

ユニークキー（取引日 + 金額 + 口座名）で既存レコードを検索し、存在する場合は更新、しない場合は新規作成します。

スキーマ検証

スクレイピング開始前に Notion DB のアクセス権・必須プロパティ・プロパティ型を自動検証し、不備があれば即座にエラーで停止します。

ℹ️ Notion API のレート制限（3 req/s）に対応するため、1 件の書き込みごとに 0.4 秒の待機を挿入しています。

5-4. GAS OTP リーダー（gas/otp_reader.gs）

Google Apps Script を Gmail に接続し、MoneyForward ID からのメール認証コードを自動取得する Web アプリです。

  • 直近 10 分以内の MoneyForward ID 認証メールを検索
  • 本文から 6 桁のコードを正規表現で抽出
  • 取得済みメールを既読にして誤取得を防止
  • シークレットトークンによるアクセス認証

5-5. GitHub Actions ワークフロー（.github/workflows/scrape.yml）

スクレイピング処理を実行する CI/CD パイプラインです。Notion Worker からの repository_dispatch と GitHub UI からの workflow_dispatch の両方をトリガーとして受け付けます。

7. セキュリティ設計

🔒 重要: MoneyForward のログイン情報・Notion トークン・GitHub PAT はすべて GitHub Secrets で管理し、コードベースには一切含まれません。

認証情報の管理

  • .env と session.json は .gitignore で除外済み
  • GitHub Actions では Secrets として注入（ログに出力されない）
  • GAS には SECRET トークンによるアクセス認証を実装

権限の最小化

  • Actions ジョブの権限は contents: read と actions: write（キャッシュ保存のため）
  • Notion Integration は必要な DB のみに共有
  • GAS は Gmail 読み取り権限のみ使用
⚠️ 本システムは MoneyForward ME の個人利用を対象として設計されています。利用にあたっては MoneyForward ME の利用規約を遵守してください。

テスト構成

ユニットテスト（37 件）

日付範囲ロジック・金額パース・日付パース・Notion スキーマ処理をブラウザ不要で検証。
make test で実行。

統合シナリオテスト（5 種）

実ブラウザ・実アカウントを使用した End-to-End テスト。make scenario で実行。S03（手動 OTP フォールバック）は手動実行のみ。

エラー時の挙動

  • スクレイピング中に例外が発生した場合、Notion ジョブページの状態を 「エラー」 に更新し、エラー詳細（最大 2000 文字）を記録します。
  • Notion DB のスキーマ不備（必須プロパティ欠落・型不一致）はスクレイピング開始前に検出し、即座にエラーで停止します。
  • GitHub Actions の実行ログ URL がジョブページに記録されるため、失敗時のデバッグが容易です。
  • GitHub Actions のタイムアウトは 15 分 に設定されています。
  • 任意プロパティ（ジョブ ID・進捗等）が Notion DB に未作成の場合、警告を出力してスキップし、処理は継続します。
💡 MoneyForward のページ構造（HTML セレクタ）が変更された場合は、scraper/mf_scraper.py 内のセレクタ定義を更新する必要があります。
